Easy-File-Utils
2026.03.26 ~ (1인 프로젝트)

📌 Summary
일상적인 파일 작업과 데이터 검수 업무를 빠르게 처리하기 위한 Python GUI 유틸리티 모음입니다.
폴더 용량 비교 도구인 checkFileSize와 Excel 파일을 CSV로 변환하는 xlsxTocsv 도구로 구성했습니다.

💡 Background
파일 복사나 백업 이후 원본과 복사본의 파일 수·용량이 일치하는지 확인하거나, 여러 엑셀 파일을 CSV로 변환하는 작업은 반복적으로 발생합니다.
명령어 사용에 익숙하지 않은 사용자도 쉽게 활용할 수 있도록 Tkinter 기반 GUI로 만들고, 경로 선택·로그 확인·결과 저장 같은 업무 흐름을 버튼 중심으로 구성했습니다.

🔍 Meaning
checkFileSize는 엑셀의 src / dest 열을 읽어 여러 폴더 쌍을 일괄 비교하고, 직접 폴더를 선택해 단건 비교도 할 수 있도록 구현했습니다.
비교 결과는 파일 수와 전체 용량 일치 여부를 색상으로 표시하며, CSV 파일로 저장할 수 있습니다.
xlsxTocsv는 단일 파일 또는 폴더 내 모든 .xlsx 파일을 재귀적으로 찾아 CSV로 변환하고, 시트 이름/인덱스와 출력 인코딩을 사용자가 지정할 수 있도록 했습니다.

⛏ Skills
Python, Tkinter, pandas, openpyxl, CSV, os, threading, ThreadPoolExecutor

⚙ Setup & Usage
git clone https://github.com/sujinbang/Easy-File-Utils.git
cd Easy-File-Utils

pip install pandas openpyxl

python checkFileSize/check_file_size.py
python xlsxTocsv/xlsxTocsv.py
Todolist-App (Daily Planner)
2026.05.26 ~ (개인 프로젝트)

📌 Summary
Todolist-App은 할 일, 루틴, 독서 기록, 다이어리 기능을 하나의 화면 흐름으로 통합한 Daily Planner 애플리케이션입니다.
React 19와 Firebase를 기반으로 사용자별 데이터를 실시간 동기화하고, Capacitor를 통해 웹과 모바일 앱 환경까지 확장할 수 있도록 구성했습니다.

💡 Background
단순한 메모나 할 일 관리 도구만으로는 복잡한 현대인의 일상을 체계적으로 관리하기 어렵다는 문제점에서 출발했습니다.
루틴화된 습관, 매일의 감정 기록, 지식 습득을 위한 독서 기록을 별도의 앱이 아닌 하나의 플랫폼에서 유기적으로 관리함으로써, 사용자가 자신의 삶의 패턴을 더 깊이 이해하고 개선할 수 있도록 설계했습니다.

🔍 Meaning
Dashboard, TodoList, RoutineManager, ReadingLog, Diary 컴포넌트를 탭 구조로 연결해 일상 관리 흐름을 한 곳에서 처리하도록 구현했습니다.
Firebase Auth 로그인 후 Firestore의 사용자별 하위 컬렉션(todos, routines, books, diaries)을 실시간 구독하며, 루틴 데이터는 오늘 날짜 기준으로 할 일에 자동 반영됩니다.
다이어리 화면은 로컬 잠금, 이미지 첨부, 감정·날씨 기록, 검색/필터링을 지원하고, Express 서버의 /api/gemini/reflect API를 통해 Ollama 기반 AI 피드백을 생성합니다.

⛏ Skills
Frontend
  • React 19 & TypeScript: 안정적이고 선언적인 UI 컴포넌트 개발
  • Vite: 빠르고 효율적인 빌드 및 개발 환경 구축
  • Tailwind CSS: 유연하고 반응형인 스타일링
  • Motion: 탭 전환 및 화면 진입 애니메이션 구현
  • Lucide React: 직관적인 아이콘 시스템
Backend & Infrastructure
  • Firebase (Firestore & Auth): 실시간 데이터베이스 및 안전한 사용자 인증
  • Node.js & Express: 일기 AI 피드백 API 서버 구축
  • Ollama (Qwen2.5): 로컬 LLM으로 일기 내용 분석 및 공감형 피드백 생성
  • Capacitor: 웹 기술을 네이티브 앱(Android/iOS)으로 변환 및 하드웨어 연동 준비
DevOps & Tools
  • Concurrently: Vite 클라이언트와 Express AI 서버 동시 실행 관리
  • ESLint & TypeScript Check: 코드 품질 및 타입 안정성 유지

⚙ Setup & Usage
필수 요구사항
  • Node.js (v18 이상 권장)
  • npm 또는 yarn
  • Firebase 프로젝트 (설정 파일 필요)
  • Ollama (AI 피드백 기능을 사용하려는 경우)
    • qwen2.5:7b 모델 설치 필요: ollama run qwen2.5:7b
설치 및 로컬 실행
1. 저장소를 클론합니다.
git clone https://github.com/sujinbang/Todolist-App.git
cd Todolist-App
2. 의존성 패키지를 설치합니다.
npm install
3. Firebase 설정 파일을 구성합니다.
- firebase-applet-config.json 파일을 프로젝트 루트에 배치합니다.

- AI 서버: http://localhost:3001
- 상태 확인: http://localhost:3001/api/health
*참고: AI 서버만 별도로 실행하려면 npm run server를 사용하세요.*

npm run build

npx cap sync
3. 각 플랫폼 개발 도구(Android Studio, Xcode)를 통해 실행합니다.
npx cap open android
npx cap open ios

📌 Summary
이슈 현황 데이터를 자동으로 분류하기 위해 문자열 기반 카테고리 분류 예측 모델을 개발했습니다.
이슈 제목이나 설명에 포함된 텍스트 데이터를 전처리하고, 머신러닝 알고리즘을 활용하여 각 이슈를 적절한 카테고리로 분류하도록 설계했습니다.
이를 통해 수작업 분류의 비효율성을 줄이고 분류 정확도를 향상시켰습니다.

💡 Background
업무 중 다양한 이슈(예: 버그 리포트, 개선 요청, 사용자 문의 등)가 텍스트 형태로 수집되는데, 이를 정확하게 카테고리화하지 않으면 우선순위나 담당자 지정에 혼선이 생겼습니다.
